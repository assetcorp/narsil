# Keyword-search engine comparison

## Run conditions

- OS / arch: Linux 6.12.76-linuxkit / aarch64
- Equal memory cap per engine: 8.6 GB
- Run depth: 1000; BM25 reference k1=0.9, b=0.4
- Same datasets, metrics, run depth, and strictly-decreasing run-file ordering for every engine.

## Engines

| Engine | Version | Ranking | Keyword setup |
|---|---|---|---|
| narsil | source (node:22-trixie-slim) | bm25 | BM25 k1=0.9 b=0.4; Narsil english analyzer (Porter stemmer, 70-word stop list) |
| elasticsearch | 9.4.2 | bm25 | BM25 k1=0.9 b=0.4 (custom default similarity); Elasticsearch `english` analyzer (Porter stemmer, English stop words) |
| meilisearch | 1.48.2 | native | Bucket-sort ranking rules (words, typo, proximity, attribute, sort, exactness), not BM25; _rankingScore for ordering; default typo tolerance and prefix search; no stemming or stop-word removal |
| opensearch | 3.7.0 | bm25 | BM25 k1=0.9 b=0.4 (custom default similarity, native Lucene BM25Similarity in 3.x); OpenSearch `english` analyzer (Porter stemmer, English stop words) |
| typesense | 30.2 | native | Native token match/proximity scoring (text_match), not BM25; english locale, Snowball stemming enabled, default typo tolerance |
